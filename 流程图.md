```mermaid
    created --> cancelled: 用户取消
    uploaded --> cancelled
    parsed --> cancelled
    analysis_completed --> cancelled
    case_completed --> cancelled

    failed --> parsing: 断点重试（解析）
    failed --> analyzing: 断点重试（分析）
    failed --> generating_cases: 断点重试（生成）

    cancelled --> [*]
```

对应表：`nb_test_tasks`（主状态）+ `nb_test_task_steps`（步骤明细，含 `timeout_seconds` / `max_retry`）

> 轮询 Worker 需检测 `running` 步骤是否超过 `timeout_seconds`；超时写入 `error_code=step_timeout` 并按 `max_retry` 断点重试或置任务 `failed`（见 [设计文档 §2.5.1](./设计文档.md)）。

## 4. AI 流水线（AiService）

设计目标为 4 个核心 Prompt 阶段（抽取 → **完备性自检** → 问题分析 → 用例生成）；**当前后端已全部实现**。

> **隐式依赖**：问题分析（Step 2）强依赖需求抽取（Step 1）的召回率；若跳过完备性自检，遗漏章节上的缺陷将无法被后续步骤发现。

```mermaid

    subgraph Input["输入"]
        SEC[document_sections<br/>结构化章节]
        TOC[文档标题 + 目录树]
    end

    subgraph Step1["Step 1: 需求抽取"]
        P1[requirement_extract.py]
        R1[ExtractRequirementsResult<br/>summary + requirements]
    end

    subgraph Step1_5["Step 1.5: 完备性自检 · 反思"]
        P15[requirement_completeness_check.py]
        R15[CompletenessCheckResult<br/>coverage_map]
        PATCH[定向补抽<br/>局部重跑 Step 1]
    end

    R2 --> P3 --> R3

    P15 --> JS1
    P2 --> JS1
    P3 --> JS1

    RETRY -->|修正 Prompt 重试| P1
```

### AI 子步骤明细（设计目标）

| 步骤 | run_type | step_name | Prompt / 模块 | 输出 | 实现状态 |
|------|----------|-----------|---------------|------|----------|
| 需求摘要 + 拆分 | `extract_requirements` | `extract_requirements` | `requirement_extract.py` | 原子需求列表 | ✅ 已实现 |
| **需求点完备性自检（反思）** | `check_requirement_completeness` | `check_requirement_completeness` | `requirement_completeness_check.py` | 覆盖映射表（`covered` / `partial` / `missing`）+ 定向补抽指令 | ✅ 已实现 |
| 清晰度/完整性/一致性/风险检查 | `analyze_requirements` | `analyze_requirements` | `requirement_analysis.py` | 问题清单 | ✅ 已实现 |
| 测试用例生成 | `generate_cases` | `generate_cases` | `case_generation.py` + `rule_context` | 结构化用例 | ✅ 已实现 |

### 4.1 完整流水线（设计目标 12 步）

与 [设计文档 §4](./设计文档.md#4-ai-流水线设计) 对齐的全链路步骤，供 `nb_test_task_steps` 编排参考：

```mermaid
flowchart TD
    S1[1. 文档解析] --> S2[2. 内容清洗]
    S2 --> S3[3. 分块切片]
    S3 --> S4[4. 需求点抽取]
    S4 --> S5["5. 需求点完备性自检 · 反思"]
    S5 --> S5a{partial / missing?}
    S5a -->|是| S4b[定向补抽]
    S4b --> S5
    S5a -->|否| S6[6. 需求关系建模]
    S6 --> S7[7. 问题检查]
    S7 --> S8[8. 问题合并]
    S8 --> S9[9. 测试设计]
    S9 --> S10[10. 用例生成]
    S10 --> S11[11. 结果校验]
    S11 --> S12[12. 导出]

    style S5 fill:#fff3cd,stroke:#856404
    style S5a fill:#fff3cd,stroke:#856404
```

**完备性自检 Prompt 引导语（摘要）**：

- 基于文档标题和目录，抽取的需求点是否覆盖了所有章节？
- 是否存在整节/整段未被抽取的模块（附录、异常流程、权限说明等）？
- 某些章节是否只抽到摘要级描述，遗漏了具体规则或验收标准？

## 10. 时序图（单次任务完整链路）

```mermaid
sequenceDiagram
    actor User as 用户
    participant FE as 前端
    participant API as 后端 API
    participant Task as 任务编排
    participant Parse as 文档解析
    participant AI as AiService
    participant QC as QualityCheck
    participant EXP as Exporter
    participant DB as 数据库
    participant LLM as LLM API

    User->>FE: 上传需求文档
    FE->>API: POST /documents/upload
    API->>DB: 创建 documents + tasks
    API-->>FE: 返回 task_id

    Task->>Parse: 解析文档
    Parse->>DB: 写入 document_sections<br/>（含 source_snapshot / parse_confidence）
    Task->>DB: 更新状态 → parsed

    Task->>AI: extract_requirements(sections)
    AI->>LLM: Prompt 1
    LLM-->>AI: JSON 结果
    AI->>DB: 写入 requirements + ai_runs

    Task->>AI: check_requirement_completeness(title, sections, requirements)
    AI->>LLM: Prompt 1.5（反思：目录覆盖自检）
    LLM-->>AI: coverage_map JSON
    AI->>DB: 写入 task_steps + ai_runs
    alt partial / missing 章节
        Task->>AI: extract_requirements(遗漏 sections)
        AI->>LLM: Prompt 1（定向补抽）
        LLM-->>AI: 补抽结果
        AI->>DB: 合并写入 requirements + ai_runs
    end

    Task->>AI: analyze_requirements(sections, requirements)
    AI->>LLM: Prompt 2
    LLM-->>AI: JSON 结果
    AI->>DB: 写入 issues + source_refs + ai_runs
    Task->>QC: ParseQualityService.check_after_analysis()
    QC-->>Task: ParseQualityReport（低置信 / inferred / 引用错乱告警）
    Task->>DB: 写入 quality_warnings JSON
    Task->>DB: 更新状态 → analysis_completed

    Task->>AI: generate_test_cases(requirements, issues)
    AI->>LLM: Prompt 3 + rules
    LLM-->>AI: JSON 结果
    AI->>DB: 写入 test_cases + ai_runs
    Task->>DB: 更新状态 → case_completed

    Task->>QC: check(requirements, cases)
    QC-->>Task: QualityCheckResult

    User->>FE: 预览 / 审阅 / 导出
    FE->>API: GET /tasks/{id}/report
    API->>DB: 查询结果
    API-->>FE: 报告 + 用例

    User->>FE: 确认导出 Excel
    FE->>API: POST /tasks/{id}/export
    API->>EXP: ExcelExporter.export()
    EXP->>DB: 写入 exports
    API-->>FE: 下载链接
    Task->>DB: 更新状态 → completed
```
